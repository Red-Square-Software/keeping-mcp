### Gaps Summary

Phase 2.5 ships a working `keeping_timer_status` tool that satisfies 9 of 10 must-have truths and all 5 ROADMAP Success Criteria as tested. The 10 D-2.5-13 unit tests pass, the full 77-test project suite passes with no regression, TypeScript is clean, Biome is clean across the three phase-modified files, and the D-2.5-09 invariant (no `src/keeping/` modifications) is preserved.

The single gap is `extractTimeEntry`'s array-shape leak (REVIEW.md WR-01). The function's own header comment promises that bare-array `time_entry` values collapse to `null` (lines 17-22, 53-56), but the implementation guard `if (candidate === null || typeof candidate !== "object")` admits arrays because `typeof [] === "object"`. The result: a drift shape `{ time_entry: [...] }` flows through as `payload.time_entry = [...]`, the opposite of the documented "strict wrapper read" contract that D-2.5-05a was written to enforce. No D-2.5-13 test exercises an array shape, so the gap is invisible from the green-bar.

D-2.5-05a is the explicit, locked rationale that exists "specifically so drift fails loudly" — an array silently passing through is the precise failure mode the decision was written to prevent. Per the verifier prompt's gap-classification instruction, this is judged a violation of D-2.5-05a rather than a non-blocking advisory.

**Closure plan (single small edit):**
1. Add `Array.isArray(candidate)` to the guard in `extractTimeEntry` (src/tools/timer-status.ts:61) so the function honours the documented "bare-array collapses to null" contract.
2. Add one or two unit tests parallel to Tests 5 and 6 covering `{ time_entry: [] }` and `{ time_entry: [{...entry}] }` → graceful-empty payload. This also addresses REVIEW.md WR-02 by giving the source comment's "schema-drift test" claim a real referent.

Optional follow-ups (not blocking, REVIEW.md IN-01..IN-03):

- Add a one-line note at the Test 4 throw site documenting the sanitised-body assumption (or wrap the literal in `sanitiseBody`).
- Decide whether to standardise mock `get` signatures across tests or note Test 9 as the sole structural guard.
