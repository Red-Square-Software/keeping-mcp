## Patterns Confirmed for Phase 3

- **Strict-wrapper extractor pattern (consolidated rule):** when a tool reads a singular-wrapper API key, the extractor guard MUST be `candidate === null || typeof candidate !== "object" || Array.isArray(candidate)`. The `typeof` check alone is insufficient because `typeof [] === "object"` in JS. Phase 3 timer write tools that need to READ entry shapes (e.g., `keeping_resume_timer` reading the last entry before resuming) should reuse this guard verbatim.
- **Drift-shape regression test pattern:** drift coverage requires BOTH a "drift collapses to null" assertion AND a "wrapped drifted data does NOT leak through" assertion. Test 12's `is_running: false` pinning despite `ongoing: true` in the array element is the structural template Phase 3 should copy.
