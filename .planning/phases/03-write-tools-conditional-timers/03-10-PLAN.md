
Regex breakdown for executor sanity:
- `^([01]\d|2[0-3])` — hour: either `00`-`19` (two digits, first is 0 or 1, second is any) OR `20`-`23`
- `:` — literal colon
- `[0-5]\d$` — minute: first digit 0-5, second digit 0-9

This rejects:
- `"1:30pm"` — `pm` suffix has no match
- `"25:00"` — hour 25 not in `[01]\d|2[0-3]`
- `"9:5"` — unpadded; `9` is one char, needs `09`; `5` is one char, needs `05`
- `"00:00:00"` — seconds segment has no match (regex ends at `$` after minutes)

And accepts (verify with positive tests):
- `"00:00"`, `"09:05"`, `"13:45"`, `"23:59"`
- The default from `nowInAmsterdamHHMM()` which always emits zero-padded HH:mm via `Intl.DateTimeFormat("sv-SE", { hour: "2-digit", minute: "2-digit", hour12: false })`.
