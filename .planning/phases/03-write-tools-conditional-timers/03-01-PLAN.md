Ship the Phase 3 write-tool foundation in one atomic vertical slice:

1. Fix `KeepingClient.rawFetch` to tolerate HTTP 204 No Content (D-3-27) — without this, `keeping_delete_entry` falsely reports SyntaxError on every successful deletion.
2. Add `KeepingClient.requestWithHeaders<T>(method, path, body?)` so timer tools can read `X-Server-Time-Ms` (D-3-18, TIMER-02). MUST share the existing `this.throttle` wrap (Pitfall 3) and the existing `pRetry` surface (writes already excluded by `shouldRetry`).

After this plan, all six write tool slices (Plans 03-02..03-07) consume these four building blocks via simple imports — no foundation work leaks into tool plans.
